full case study

InternTrack keeps every internship step visible.

A product-style frontend concept for students who need one calm place to track applications, interview prep, follow-ups, and portfolio tasks.

View live project Project source Back to case studies
Project type Dashboard UI concept
Audience Students applying for internships
Focus Pipeline clarity and prep rhythm
Build Static HTML and CSS
problem

Applications become messy when the next action is not obvious.

Students often split internship tracking across notes, screenshots, calendar reminders, and chat messages. The important question gets buried: which opportunity needs action today?

goal

Turn scattered application work into a clear operating screen.

The goal was to make status, prep work, and follow-up timing visible at the same time, while keeping the page light enough to read quickly on a portfolio review.

design decisions

Three choices that made the concept useful.

The page practices dashboard information architecture without turning into a dense spreadsheet.

  1. 01

    Pipeline lanes come before metrics.

    The board starts with application stages so the user can understand movement first. Numbers support the story, but they do not replace the work items.

  2. 02

    Prep tasks sit next to real opportunities.

    Interview notes, portfolio updates, and follow-up drafts are treated as part of the application flow, not as separate chores below the dashboard.

  3. 03

    Visual hierarchy stays calm and scannable.

    Soft surfaces, blue status accents, clear spacing, and compact tags help the page feel organized even when several companies and tasks appear at once.

reviewer signal

The work shows more than a pretty screen.

InternTrack is useful in the portfolio because it shows how a UI can organize a real workflow: what is happening, what matters next, and how the user keeps momentum.

result

A reviewer can understand the product idea without needing a demo account.

The finished page works as a static but believable dashboard concept. It communicates the user problem, the interface structure, and the frontend practice behind the build.

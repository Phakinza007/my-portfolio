# Product

## Register

brand

## Users

This portfolio is for visitors who need to understand the work quickly: hiring reviewers, teachers, classmates, collaborators, and the portfolio owner maintaining practice projects over time.

Visitors arrive with limited context. They should be able to scan the cover page, see that each project is real, open a live page, and inspect the source without hunting through the repository.

## Product Purpose

Portfolio Cover Gallery turns a folder of standalone HTML practice files into a professional, browsable frontend portfolio. The root `index.html` is both a visual cover and a clear navigation layer for every project.

Success means the portfolio feels like a real public site, not a loose file dump. Each project should have a useful thumbnail, a direct live link, a source link, and enough context for a reviewer to understand what kind of frontend practice it represents.

## Brand Personality

Warm, practical, polished, and learning-forward.

The voice should feel confident without pretending the work is a finished commercial product. It should show curiosity, care, and improvement. Thai-first copy is welcome where it helps the page feel personal and local; English labels are appropriate for common portfolio actions such as "View live" and "Source file".

## Anti-references

Avoid generic template galleries where every card looks interchangeable. Avoid empty thumbnail grids, placeholder blocks, fake project previews, and decorative sections that do not help visitors open or understand the work.

Avoid over-decorated AI-looking landing pages: excessive gradients, vague feature copy, floating cards for their own sake, and visual effects that distract from the actual projects. Do not hide that this is a practice portfolio. The learning context is part of the value.

## Design Principles

1. Show real work first. Use screenshots and live links as the main proof of quality.
2. Make every project openable. A visitor should never wonder whether a card is decorative or clickable.
3. Keep source transparency. Live project links and repository source links should stay visually distinct.
4. Polish the presentation, not the truth. The portfolio can feel professional while still being honest about practice work.
5. Keep the page maintainable. New projects should be easy to add without redesigning the whole system.

## Accessibility & Inclusion

Use semantic HTML, meaningful landmarks, descriptive image alt text for project screenshots, and visible keyboard focus states. Touch targets should remain comfortable on mobile.

Respect reduced-motion preferences when adding motion. Maintain strong text contrast across warm neutrals and accent colors. Keep Thai and English copy readable, concise, and free of cramped line lengths.

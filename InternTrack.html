full case study

From scattered applications to one calm workflow.

This page is not only a dashboard mockup. It explains the problem, goal, design decisions, practiced skills, and result so a reviewer can understand the thinking behind the interface.

Problem

Internship search work gets split across notes, email, calendar, and portfolio tasks.

A student can miss follow-ups or walk into interviews unprepared when application status, prep notes, and portfolio checks live in different places.

Goal

Create a fast-scanning workspace for applications, prep, and next actions.

The interface should make progress visible in seconds, keep interviews tied to prep work, and feel simple enough to check from a phone before class or a meeting.

01

Pipeline first, because status is the main question.

Applied, Interview, and Follow-up lanes create an immediate map of the search. This avoids a heavy spreadsheet feeling while still showing what needs attention.

02

Prep tools stay beside the board instead of becoming a separate page.

The interview calendar and checklist sit next to applications, so portfolio checks, case-study notes, and follow-up drafts feel connected to real opportunities.

03

Compact tags carry urgency without adding long copy to every card.

Tags like portfolio sent, reply soon, task prep, and send email keep each application scannable while still giving enough context for the next move.

What I Practiced

InternTrack was built to show product thinking as much as visual polish. The page practices app-style layout, responsive dashboard behavior, and clear UX writing.

dashboard UI responsive CSS UX writing semantic HTML visual hierarchy case-study storytelling

Result

The final page gives the portfolio a practical product concept that reviewers can inspect, open from the homepage, and understand without extra explanation.

  • Shows a real student workflow instead of a generic dashboard shell.
  • Combines status, interview prep, and follow-up tasks in one responsive layout.
  • Documents the design reasoning so the project reads like reviewer-ready portfolio work.
